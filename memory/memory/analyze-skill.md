# /analyze 命令

## 触发
- 用户输入 "/analyze" 或 "分析一下" 或 "分析本轮"
- 对话中手动触发，不自动运行

## 输出格式

```
📊 本轮认知侧写

你的逻辑模式：
- [怎么思考的，有什么变化]

你的性格表现：
- [这轮暴露了什么新特征]

配合问题：
- [我哪没做好]
- [你哪卡住了]

🔧 我的行为调整（下次立刻生效）
- [具体规则改动]
```

## 执行流程
1. 扫描本轮对话
2. 输出分析报告
3. 询问："是否保存到 user-profile？是否执行行为调整？"
4. 用户确认 → 写入 memory + 推送 GitHub

## 规则
- 有新的发现才分析，没有就跳过（不要硬编）
- 不重复已有的认知
- 行为调整必须具体可执行，不能是空话
- 分析结果只保留最新 3 次在 profile 中

**Why:** 用户手工触发深度分析，掌控节奏。hook 只管静默备份。
**How to apply:** 用户输入 /analyze 时执行。
