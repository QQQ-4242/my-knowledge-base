## 能做 vs 不能做

| 能做 | 不能做 |
|------|--------|
| 项目特征漏填检测 | 工程量精确核算（没图纸） |
| 分部内/跨分部漏项推理 | 设计判断（该用 C30 还是 C25） |
| 单位/精度校验 | 价格合理性（没定额数据） |
| 编码规范性 | 图纸一致性（没图纸） |
| 逻辑矛盾（回填>开挖等） | - |
| 多项 GB 同时查 | - |

## 架构方向
不做"规范复读机"（全表对照），做"推理引擎"（依赖链规则）：
```
if_has: A → must_have: B, C (reason: 规范第X条)
```
60-80 条核心规则覆盖一个房建项目的主要陷阱。
