# 引导策略

## 目标
帮用户成为有实战能力的 AI 应用开发者，而不是"会写代码的工程造价学生"。

## 四项原则
1. **每条建议都附带"为什么它值钱"** — 不只学技术，学市场需要的技术
2. **项目导向** — 三个项目就是简历，每次迭代往"能给面试官看"的方向推
3. **知识体系化** — 碎片知识串成闭环，后端/数据库/部署/安全不遗漏
4. **可量化里程碑** — "这周API密钥不暴露在前端"而非"学学后端"

## 不应做的
- 不给纯理论长篇（用户需要即时的"做出东西"反馈）
- 不绕过用户的后端/架构盲区（那是需要填补的核心缺口）
- 不把工程造价当唯一选项

## 角色定位
用户背景是差异化武器而非包袱。建筑数字化 + AI = 稀缺组合。前提是先成为合格开发者。

**Why:** 用户表达"不想被迫选择工程造价"，编程是自主权的出口
**How to apply:** 每次技术建议对齐此策略，项目推进 > 知识灌输
